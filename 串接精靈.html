里民小幫手 串接精靈

這份精靈不是舊版那種只填欄位的表單，而是把目前正式版真正需要的 GitHub、Google、Apps Script、LINE、Make 串接資訊整理成可交接的設定流程。填完後可直接匯出設定檔與 Apps Script 常數區塊。

先複製模板不要直接改母版資料夾，先複製成新的里別專案後再填。
現行架構已更新現在是單一 Apps Script、單一 Google Sheet、三個照片資料夾、兩條 Make 通知 Webhook。
精靈用途幫你蒐集資料、提醒去哪裡拿金鑰、產出 `config.json` / `config.js` / `storeconfig.js` / `apps-script-constants.gs`。

1. 基本資訊

先把新里別的識別資訊填好，後面網址、檔名與預設代號都會自動帶入。

新專案資料夾路徑 必填
區名 必填
里名 必填
系統代號 code 必填
建議用英文大寫。GitHub Repo 名稱預設會跟它一樣。
案件前綴 case_prefix 必填
Apps Script 送件編號會用到，例如 `TP260422001`。
完整里名 village_name 自動
系統名稱 system_name 可改
為什麼現在多了 case_prefix？

現行版 Apps Script 在建立案件編號時，不應再固定用舊社里的 `HP`。所以模板已改成用可設定的 `CASE_PREFIX`，讓每個新里別可以用自己的前綴。

2. GitHub 與網站網址

GitHub Pages 仍然是網站前台與後台頁面的部署方式。

GitHub 帳號 必填
Repo 名稱 自動
Pages URL 自動
BASE_URL 自動
GitHub Pages 目前應該怎麼設？
  1. 建立 Public repository。
  2. 到 Repo 的 `Settings → Pages`。
  3. Source 選 `Deploy from a branch`。
  4. Branch 選 `main / root`。
  5. 幾分鐘後網站會出現在 `https://帳號.github.io/Repo名/`。

3. Google Sheet 與 Drive

現行版至少需要 1 個 Google Sheet 與 3 個照片資料夾。

Google Sheet ID 必填
Drive 主資料夾 ID 建議填
NBH 資料夾 ID 必填
STOR 資料夾 ID 必填
BULLETIN 資料夾 ID 必填
建議 Drive 結構：`專案根資料夾 / Photos / NBH、STOR、BULLETIN`。Google Sheet 可以放在根資料夾同層。
工作表目前要有哪些？
  • `案件清單`：30 欄，含公開欄位與置頂順序。
  • `商店清單`：32 欄，含方案類型與置頂順序。
  • `管理員名單`：新版 4 欄（顯示名稱 / 角色 / 啟用 / Email）。
  • `公佈欄`：8 欄；若未建立，可在 Apps Script 手動執行 `setupBulletinSheet()`。
欄位標題快速複製
案件清單（第 1 列貼上）
商店清單（第 1 列貼上）
管理員名單（第 1 列貼上）
公佈欄（第 1 列貼上）

4. Apps Script 與 Google OAuth

後台登入與所有資料讀寫，現在都走同一支 Apps Script。

Apps Script Web App URL 必填
Google OAuth Client ID 必填
API Key（商店上傳照片會用到） 自動產生
Apps Script 部署後第一件事要做什麼？
  1. 把模板資料夾內的 `apps-script.gs` 貼到 Google Apps Script。
  2. 先用精靈匯出的 `apps-script-constants.gs` 覆蓋 `TEMPLATE CONSTANTS` 區塊。
  3. 部署成 Web App：執行身分選自己，存取選所有人。
  4. 第一次請手動執行：
    • `setupPinColumns()`
    • `setupBulletinSheet()`
Google OAuth Client ID 要去哪裡拿？
  1. 到 Google Cloud Console 建立專案。
  2. 建立 OAuth Client ID，類型選 Web application。
  3. Authorized JavaScript origins 至少加上 GitHub Pages 網址。
  4. 把 Client ID 貼進這裡；後台頁面 `admin.html` / `bulletinlist.html` / `storelist.html` 會用到。

5. LINE 官方帳號與 Make 通知

目前正式版是由 Apps Script 呼叫 Make Webhook，再由 Make 去推送 LINE 通知。

案件通知 Webhook 必填
商店通知 Webhook 必填
LINE 官方帳號名稱 建議填
LINE Basic ID 建議填
LINE Channel ID 建議填
LINE 群組 ID 日後擴充
LINE Channel Secret 交接紀錄
LINE Channel Access Token 交接紀錄
提醒：目前這些 LINE 金鑰不會直接寫進公開網站前端。精靈保留這些欄位，主要是為了交接紀錄與未來擴充，不是叫你把秘密金鑰公開上 GitHub。
LINE 官方帳號與 Make 的建議設定順序
  1. 到 LINE Official Account Manager 建立官方帳號。
  2. 到 LINE Developers 建 Provider 與 Messaging API Channel。
  3. 保存 Channel ID / Secret / Access Token。
  4. 在 Make 建兩條流程：
    • 新案件通知
    • 新商店申請通知
  5. 把 Make Webhook URL 貼回這份精靈。

6. 管理員與交接備註

這裡的 Email 會進 `config.json`，方便交接；真正權限仍要寫入 Google Sheet 的管理員名單。

管理員 Email（每行一個） 建議填
目前 Apps Script 的管理員維護功能只允許 super admin。建議在 Google Sheet「管理員名單」中，至少先建立一位角色為「超級管理員」且啟用為 TRUE 的帳號。
現行版重要檔案
config.js案件 / 公告 / admin 頁面共用設定。
storeconfig.js商店頁面設定；`store.html` 的照片上傳會用到 API Key。
apps-script.gs統一後端；模板內這份已經改成現行架構占位版。
assets/no-photo.svg案件無圖時的預設圖片，Apps Script 也會引用這個 BASE_URL 路徑。

7. 匯出設定檔

建議至少匯出四個檔案並放回你剛複製的新專案資料夾。

8. 建議操作順序

        

          
1.先建立 GitHub Repo 與 GitHub Pages。

          
2.建立 Google Drive 資料夾與 Google Sheet。

          
3.填入 Google OAuth Client ID。

          
4.把模板的 `apps-script.gs` 貼到 Google Apps Script，並部署成 Web App。

          
5.把部署後的 `script_url` 貼回精靈。

          
6.設定 Make 與 LINE 官方帳號通知，填入兩條 Webhook。

          
7.匯出 `config.js` / `storeconfig.js` / `config.json` / `apps-script-constants.gs`。

          
8.用正式版 `舊社里小幫手` 為基底建立新專案並覆蓋設定檔。
